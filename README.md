# Soccer Pinball '98 — в браузере

Запуск классической игры **Dynamite Soccer Pinball '98** (Pin-Ball Games Ltd., 1998,
Windows 95/98) прямо в браузере — без установки, целиком на стороне клиента.

### ▶ Играть онлайн: **https://avanturist888.github.io/pinball-soccer-98/**

Оригинал распространялся как единый Windows‑исполняемый файл (DirectDraw‑игра).
Здесь он запускается через [**retrowin32**](https://github.com/evmar/retrowin32) —
эмулятор win32 (свой интерпретатор x86, без Wine и без Linux), скомпилированный
в WebAssembly. Это **эмуляция**, а не декомпиляция/переписывание: движок игры
(`winpin.exe`) исполняется как есть, а Windows‑API (DirectDraw / DirectSound /
DirectInput / WinMM) реализованы на стороне эмулятора.

Полный цикл работает в браузере: интро → выбор языка → выбор стола → NEW GAME →
выбор команды → игровое поле. Плунжер запускает мяч, флипперы отбивают, звук идёт
через Web Audio. Прогресс (сейвы/рекорды) сохраняется в `localStorage`.

## Играть

Открой `docs/index.html` через любой статический веб‑сервер (или задеплой на GitHub
Pages). Локально:

```bash
cd docs && python3 -m http.server 8000
# затем открой http://localhost:8000/
```

> ⚠️ Нельзя открывать `index.html` как `file://` — нужен HTTP‑сервер
> (страница грузит `.wasm` и файлы игры по сети через `fetch`).

Никаких особых заголовков (COOP/COEP/SharedArrayBuffer) **не требуется** — сборка
однопоточная и работает на любом статическом хостинге, включая GitHub Pages.

### Управление с клавиатуры (по умолчанию, меняется в игре: OPTIONS → SETUP)
| Клавиша | Действие |
|---|---|
| **Z** | левый флиппер |
| **Right Shift** | правый флиппер |
| **Enter** | запуск мяча (плунжер) |
| **Space** | толчок стола вверх |
| **X** | толчок вправо |
| **/** (Slash) | толчок влево |

Кнопки в меню — мышью (на тач‑устройствах — тапом).

### Управление на телефоне
На тач‑устройствах под игрой автоматически появляется экранная панель кнопок:
**◀ ФЛИППЕР** (Z), **ФЛИППЕР ▶** (Right Shift), **ПОДАЧА** (Enter),
**ТОЛЧОК** (Space). Кнопки поддерживают удержание и одновременное нажатие двух
флипперов. Панель показывается только при сенсорном вводе; на десктопе остаётся
подсказка с клавишами.

## Что внутри (`docs/`)
| Файл | Назначение |
|---|---|
| `index.html` | страница игры: загрузчик, canvas, подсказки/тач‑управление |
| `soccer.bundle.js` | склеенный клиент (загрузка файлов, ввод, аудио, монтаж canvas) |
| `wasm.wasm` | эмулятор retrowin32, скомпилированный в WebAssembly (~2 МБ) |
| `game/` | файлы игры (765 файлов, ~78 МБ) + `manifest.json` — их список |
| `.nojekyll` | отключает Jekyll на GitHub Pages |

При старте клиент читает `game/manifest.json` и параллельно (24 потока `fetch`)
загружает все файлы игры в виртуальную файловую систему эмулятора в памяти, затем
запускает `winpin.exe`.

## Деплой на GitHub Pages
1. Запушь репозиторий на GitHub.
2. Settings → Pages → Source: **Deploy from a branch**, ветка `main`, папка **`/docs`**.
3. Через минуту сайт будет доступен по адресу `https://<user>.github.io/<repo>/`.

Ограничения Pages соблюдены: макс. файл ~2 МБ (< 100 МБ), весь сайт ~80 МБ (< 1 ГБ).

## Как это собрано

Рантайм собирается из форка retrowin32
([`avanturist888/retrowin32`](https://github.com/avanturist888/retrowin32),
ветка `soccer-pinball-input`) — туда дописаны DLL‑заглушки и фиксы эмуляции,
без которых игра не грузилась (DirectDraw Blt/EnumDisplayModes, WinMM mmio для
WAV, DirectSound‑микшер, `SetCurrentDirectoryA`, файловая система без учёта
регистра и запись сейвов). Пересборка и стейджинг в `docs/`:

```bash
./tools/deploy_retrowin32.sh
```

Скрипт: собирает wasm (nightly Rust), бандлит TypeScript‑клиент (esbuild),
копирует `wasm.wasm` и `index.html`, раскладывает файлы игры в `docs/game/` и
генерирует `manifest.json`.

Извлечение файлов игры из оригинального инсталлятора описано в
[`docs-build/HOWITWASMADE.md`](docs-build/HOWITWASMADE.md) (распаковка
Gentee‑инсталлятора, определение графического стека). Сам рантайм там относится
к прежней сборке на BoxedWine — текущая версия работает на retrowin32.

## Известные ограничения
- **Первая загрузка стола не мгновенна** — эмулятор интерпретирует x86 (в
  web‑сборке нет JIT), а распаковка ассетов (`gfx.wad`) тяжёлая. Загрузка разовая,
  дальше игра идёт из памяти.
- **Экран выбора языка кликается не сразу** — он анимируется (флаги/фейд) и
  становится интерактивным только после завершения анимации; на wasm это заметно
  медленнее нативного. Это тайминг, а не баг.
- **Сейвы — в `localStorage`** конкретного браузера (синхронное хранилище под
  синхронную ФС эмулятора). Очистка данных сайта их сотрёт.
